## 1. 产品定位

- YIAI Center 是一个领域无关的 AI 能力装配、运行解释、质量治理和成本治理演示平台。
- 产品用于 AI 产品经理面试，重点是证明产品判断和完整闭环。
- 平台可以保存多个垂直 Agent。
- 一次用户请求只允许 Router 选择并运行一个垂直 Agent。
- 主垂直 Agent 可以使用已绑定并随 Release 发布的自然语言 Skill、文本 RAG、远程只读 MCP Tool 和预置业务读写 Tool。
- 不建设多 Agent 讨论、委派、竞争、投票或联合规划。
- 不把平台设计成物业、医保、商场或其他行业业务系统。

### 1.1 演示项目边界

- 这是个人求职面试演示项目，优先完成真实、可解释、可操作的产品闭环。
- 只保留避免密钥泄露、误删演示数据和重复写入所必需的最小保护。
- 不把登录、权限、安全中心、监控告警平台、备份中心和容灾中心包装成产品能力。
- 不为模拟生产环境增加高可用、自动扩缩容、灰度发布、蓝绿发布和复杂审批流程。
- 不因生产级安全运维要求阻塞面试演示功能。

## 2. 行业无关

- 行业只出现在产品负责人面试演示时的口头介绍和手工替换内容中。
- 产品负责人可以手工替换 Agent 说明、Skill、RAG、MCP 连接、预置 Tool 文案和演示数据。
- 替换内容后，通过 Release 发布新的演示版本。
- 平台中不预设任何行业名称、行业 Logo、行业示例问题和行业图标。
- 不建设行业模板、行业选择器、术语切换和换肤系统。
- 不提供“一键切换物业／医保／商场”的功能。

## 3. 顶层页面与身份

- 页面只有三个顶层 TAB：用户、员工、平台管理。
- 三个 TAB 是产品导航，不是登录身份。
- 用户 TAB 展示 AI 对话和我的工单。
- 员工 TAB 展示人机共驾和工单处理。
- 平台管理 TAB 展示配置、发布、运行、质量和成本治理。
- 不建设登录、注册、退出和忘记密码。
- 不建设身份切换器。
- 不建设用户、员工、管理员账号。
- 不建设角色、权限和授权配置页面。

## 4. 四个面试故事

### 4.1 可解释运行

- 每条用户消息创建一个独立 Run。
- 每个 Run 显示 Release、Router、垂直 Agent、Skill、RAG、MCP/Tool、模型、用量、延迟、成本和最终状态；“垂直 Agent”是页面名称，“每个 Run 只能有一个”是运行规则。
- 每条用户消息和 AI 回答都显示时间戳。
- 用户页面展示历史对话列表；每项至少显示对话标题、最近消息时间和消息数，点击后恢复该对话。
- 每个 AI 回答气泡下方直接显示 Run 状态、垂直 Agent、Release、时间和“查看 Run 详情”入口。
- 从 AI 气泡打开 Run 详情时使用右侧抽屉，不要求用户先切换到平台管理。
- Run 详情明确展示本次用户输入和客服最终回答；新 Run 的输入与回答同时进入 Trace。
- 用户能看到本轮回答所引用的 RAG 证据。
- 用户能看到 Tool 是否调用、是否成功和结果摘要。
- 不展示隐藏思维链。
- 不用虚假 Tool 结果、虚假引用和虚假成本填充页面。

### 4.2 版本化发布

- Agent、Skill、RAG、远程只读 MCP Tool 绑定、预置 Tool 绑定和运行成本策略都通过 Release 发布。
- 流程是：编辑草稿 → 校验 → 创建候选 Release → 查看差异 → 人工发布。
- 发布后创建的下一条新消息使用新 Release。
- 在旧会话中继续发送的新消息也使用当时最新的 Active Release。
- 已经生成的旧气泡、旧 Run 和旧 Trace 保持原 Release，不重新计算。
- 新建会话使用当时最新的 Active Release。
- 保留历史 Release。
- 可以人工回滚；回滚后创建的新消息使用被恢复的 Release。
- 保存草稿不能直接影响在线运行。
- 不自动发布和自动回滚。

### 4.3 Badcase 治理

- 系统可以自动抓捕疑似 Badcase。
- 用户点踩可以生成疑似 Badcase。
- 疑似 Badcase 必须经过人工确认才成为正式 Badcase。
- Darwin 生成诊断草稿和原因分类建议。
- 人工确认根因、修改方案和最终关闭。
- 正式 Badcase 可以生成 Evaluation Case 草稿。
- 人工发起新旧 Release 回放比较。
- 新 Release 发布后回放原始 Badcase，再由人工关闭。
- Darwin 不自动修改 Agent、Skill、RAG、MCP、Tool 和 Release。
- Evaluation 不自动发布。

### 4.4 成本追踪和控制

- 普通 Router、垂直 Agent 和其他常规云模型调用统一使用 DeepSeek V4-Flash，并开启思考模式。
- 只有用户明确发起“高级专家”工作流时才允许使用 DeepSeek V4-Pro，并开启思考模式；示例是 Badcase AI Darwin。
- 每次云调用必须显示实际使用的模型，因此可以证明普通请求没有误用高成本模型。
- 普通请求、低置信度或模型回答失败时，不自动升级到 V4-Pro。
- Trace 路径中每一次云模型 API 调用都有独立 Snap 详情。
- CloudCallSnap 直接嵌在对应 Trace 云调用完成步骤中，不把成本事实拆到无关联的独立区域。
- 每次模型调用展示输入未命中缓存、输入命中缓存和输出 Token。
- 每次模型调用展示 Provider、模型、调用阶段、时间、延迟、人民币单价快照和人民币 Estimated Cost。
- Run 详情底部展示本 Run 的调用次数、三类 Token、总延迟和人民币总成本。
- DeepSeek 官方美元原价作为来源信息保留；面向用户的价格和成本统一按人民币显示，并展示演示汇率快照。
- Run 展示模型调用次数、RAG 召回与注入量、MCP 调用与结果长度、预置 Tool 调用次数和总延迟。
- 可以按 Run、Release、Agent、模型和时间查看成本。
- 可以配置上下文长度、RAG 条数、MCP 次数、模型调用次数和输出 Token 等限制。
- 可以配置单 Run 和单日 Estimated Cost 预警。
- Evaluation 同时比较质量、Token、延迟和 Estimated Cost。
- 云模型已经返回答案但用量字段异常时，答案仍向用户展示。
- 用量字段异常时明确显示“用量异常”，自动生成疑似 Badcase，不编造数字。
- 不以“拿不到用量”为理由拒绝展示已经生成的答案。
- 不猜测、补零或伪造缓存命中、缓存未命中和输出 Token。
- 不根据成本自动切换模型、修改配置或发布。

## 5. Agent

- 系统只有一个 Router。
- Router 可以配置多个候选垂直 Agent。
- Router 每个 Run 只能选择一个垂直 Agent。
- 默认提供一般客服、投诉客服和工单处理三个垂直 Agent。
- 平台管理可以新建、编辑、校验和停用垂直 Agent。
- Agent 可以绑定 Skill、RAG、远程只读 MCP Tool 和预置业务 Tool。
- 低置信度时可以澄清、使用兜底 Agent或建议人机共驾。
- 一个 Run 不允许依次尝试多个垂直 Agent。

## 6. Skill

- Skill 是产品负责人能够直接阅读和修改的自然语言作业指导书。
- 支持在页面直接创建和编辑 Skill。
- 支持从公开 Git URL 导入 Skill。
- Git 导入时展示固定版本、SKILL.md 内容、文件清单和安全检查结果。
- 导入后仍需校验、绑定 Agent 并进入 Release。
- Skill 页面展示名称、说明、适用条件、不适用条件、执行步骤和输出要求。
- 不执行 Skill 中的脚本。
- Git 仓库只要发现脚本、可执行文件或脚本执行要求，就拒绝导入并说明原因。
- 不自动跟随 Git 主分支更新。
- 不允许 Skill 绕过 Release 直接上线。

## 7. 文本 RAG

- V1 只支持粘贴纯文本或 Markdown。
- 接入流程是：粘贴文本 → 填写名称、标签和版本 → 预览切分 → 建立关键词和向量索引 → 检索测试 → 绑定 Agent → 加入 Release → 人工发布。
- 切分预览展示切分模式、切分版本和实际技术名称。
- 索引状态展示关键词检索方式、向量模型、向量检索方式和融合方式。
- 检索测试展示命中片段、相关度和最终引用。
- V1 上线时提供 3 篇领域无关的长文档。
- 三篇文档分别覆盖通用服务规则、通用工单规则和通用问题处理指引。
- 三篇文档不能出现物业、医保、商场等行业迹象。
- 不支持 PDF、Word、Excel、图片、OCR、网页抓取和批量同步。
- 不生成没有来源的假引用。

## 8. MCP 与预置业务 Tool

### 8.1 远程只读 MCP

- 平台只连接已经部署并正在运行的远程 MCP Server。
- 页面支持填写 Endpoint、测试连接、查看 Server 信息、查看 Tool 和参数、单 Tool 测试、绑定 Agent 和发布。
- 平台管理人工选择允许使用的查询类 Tool。
- 远程 MCP Tool 只提供读取和查询能力。
- 不从 Git 下载、构建和部署 MCP Server。
- 不在平台中运行 MCP 源码、启动命令和本地进程。
- 不接入远程动态写 MCP。

### 8.2 预置业务读写 Tool

- 预置业务 Tool 由项目代码提前定义。
- 可以通过 Agent 绑定决定某个 Release 是否提供这些 Tool。
- 工单 Agent 可以查询工单列表。
- 工单 Agent 可以读取工单详情。
- 工单 Agent 可以生成创建工单草稿。
- 工单 Agent 可以生成更新工单草稿。
- 工单 Agent 可以生成关闭工单草稿。
- 工单 Agent 可以生成删除工单草稿。
- 创建、更新、关闭和删除都必须先展示确认卡。
- 用户确认后执行并显示结果回执。
- 未经确认不能执行写操作。
- 不允许模型直接跳过确认。

## 9. 工单

- 用户 TAB 的“我的工单”显示当前演示数据中的用户工单。
- 员工 TAB 可以查看和处理全部演示工单。
- 工单包含编号、主题、描述、类别、优先级、状态、处理结果和更新时间。
- 创建工单需要确认。
- 更新描述、类别、优先级或补充信息需要确认。
- 关闭工单需要确认。
- 删除工单需要二次确认，并明确提示删除影响。
- 每次写操作显示成功、失败或结果不确定。
- 结果不确定时不自动重复执行。

## 10. 人机共驾

- 产品统一使用“人机共驾”，不使用“人工接管”作为核心名称。
- 用户可以请求人工协助。
- AI 可以因为低置信度、能力失败或连续未解决而建议人工协助。
- 员工可以查看完整对话、AI 摘要、已使用能力、未完成动作和当前成本摘要。
- 员工回复一次后，可以交给 AI 继续或结束会话。
- AI 恢复后读取人工回复和处置摘要。
- 产品数据允许未来多次人机往返。
- V1 演示只验收一个人工回复和一次 AI 恢复。
- 人工输出期间 AI 不能并发回复。
- 用户主动请求人工不自动成为 Badcase。
- 不建设排队、抢单、分配、转派、技能组、SLA 和绩效。

## 11. 疑似 Badcase 自动抓捕

### 11.1 自动抓捕条件

- Run 以 error 结束。
- Router 没有合法结果或置信度低于阈值。
- Skill、RAG、MCP 或预置 Tool 失败并影响回答。
- 工单写操作失败或结果不确定。
- 最终回答为空或不符合约定格式。
- AI 因失败、低置信度或连续未解决进入人机共驾。
- 用户点踩。
- 任一云模型调用缺少必须的三类 Token 用量。
- 成本限制导致回答降级。
- 单 Run 成本或延迟超过异常阈值。
- 运行记录缺少完整终态或关键步骤。

系统只负责生成疑似记录和分类建议，不自动确认根因。

### 11.2 一级与二级分类

一级一：运行与模型

- 模型调用失败。
- 调用超时。
- 流式输出中断。
- 最终回答为空。
- 用量信息不完整。
- Trace 不完整。

一级二：Router 与 Agent

- 无路由结果。
- 路由低置信度。
- 路由格式错误。
- Agent 选择疑似错误。
- Agent 边界疑似错误。

一级三：Skill 与 RAG

- Skill 应触发未触发。
- Skill 不应触发却触发。
- Skill 指令未被遵守。
- RAG 无召回。
- RAG 召回低相关。
- 引用缺失。
- 引用错误。
- 索引异常。

一级四：MCP、Tool 与工单

- MCP 连接失败。
- MCP Tool 不存在或参数变化。
- MCP 调用失败。
- 预置 Tool 调用失败。
- 写操作确认异常。
- 写操作失败。
- 写操作结果不确定。

一级五：回答质量

- 用户点踩。
- 回答不完整。
- 回答与证据冲突。
- 回答未解决问题。
- 回答格式不符合要求。

一级六：人机共驾

- 不应转人工却转人工。
- 应转人工未转人工。
- AI 交接信息不完整。
- 人工交回后 AI 恢复失败。

一级七：成本与性能

- 单 Run Token 异常。
- 缓存命中异常。
- 模型调用次数异常。
- RAG 上下文过长。
- MCP 结果过长。
- Estimated Cost 异常。
- 延迟异常。
- 成本限制导致质量下降。

自动分类只是建议，正式一级和二级分类由人工确认。

## 12. 平台管理页面

- 总览：Active Release、运行状态、Run、成功率、Badcase、人机共驾和成本摘要。
- Agent 中心：Router 和垂直 Agent。
- 能力中心：Skill、RAG、远程 MCP 和预置 Tool。
- Release 中心：草稿、候选、差异、发布、历史和回滚。
- Run 与 Trace：问题、回答、完整路径、每次云 API Snap、用量、成本和异常。
- Run 与 Trace 的列表、事件、Snap、输入、回答、Release 创建/发布时间都显示时间戳。
- 成本：趋势、构成、高成本 Run、预警和控制策略。
- 质量：疑似 Badcase、正式 Badcase、Darwin、Eval Case 和回放。
- 系统设置：DeepSeek、单价、连接状态和演示数据。
- 不建设装饰性大屏。

## 13. V1 最小演示验收

- 三个顶层 TAB 可直接切换，无登录和身份切换。
- 普通咨询、投诉和工单请求命中三个不同 Agent，且每个 Run 只有一个 Agent。
- Git Skill 无脚本时可导入；发现脚本时明确拒绝。
- 三篇通用长文档可以切分、索引、检索、引用和发布。
- 一个已部署远程只读 MCP 可以连接、查看 Tool、测试、绑定和调用。
- 工单可以查询、创建、更新、关闭和删除；所有写操作都经过确认。
- Release 发布后，所有新消息使用新 Release；旧气泡保持不变。
- 人机共驾完成一个人工回复和一次 AI 恢复。
- 自动规则可以生成疑似 Badcase，并给出一级和二级分类建议。
- Badcase 可以经过诊断、人工修改、Evaluation、发布和原案例复验后关闭。
- 每次云模型调用都显示三类 Token；异常时不拦截答案、不编造数据。
- 用户可以从历史对话中的 AI 气泡打开右侧 Run 详情，并看到输入、客服回答、完整 Trace、逐步人民币成本和 Run 汇总。
- 可以按 Run、Release、Agent 和模型查看 Estimated Cost。
- 可以演示一次成本限制和一次成本预警。

## 14. 文档维护规则

- 每个开发任务开始前，Codex 必须读取本文件、架构说明和版本规划。
- Codex 必须先复述本次涉及的产品要求，再开始编码。
- 每个小版本完成后，五份文档使用同一个 V0.5.x 版本号。
- 本文件只在产品规则发生变化时修改正文。
- 即使正文没有变化，也更新“已复核版本”和更新时间。
- 版本规划记录完成状态和真实证据。
- 产品负责人可以随开发过程新增、删除或调整产品判断。
- 真实开发证据与旧文字冲突时，Codex应先指出冲突并提出文档更新建议。
- 已规划功能可以根据成本、风险和验证结果拆分、合并、延期或取消。
- 不把临时 Bug 过程、代码细节和接口字段堆进产品说明。
- Codex 不得在未说明原因和影响的情况下自行改变产品判断。

## 15. V0.5.9 远程只读 MCP 补充判断

- 平台可以同时保存多个已经运行的远程 MCP Server，并展示来源、固定版本、Endpoint、Transport、鉴权、状态、Tool 清单、读写属性、白名单、Agent 绑定和 Release 状态。
- 产品负责人可以在平台中测试连接、选择只读 Tool、绑定或解绑 Agent，再通过 Candidate Diff 和人工发布使变更生效。
- 发布后只影响下一条新消息；历史 Run 继续显示当时的 Server、Endpoint、固定版本、Tool、参数、结果摘要和 Release。
- 首批演示包含独立自部署的命语计算型只读 MCP 和外部 MCP 官方文档 Server。
- 命语演示 Release 只允许 `ziwei_calculate`；排盘请求缺少性别或历法时只集中追问一次，不猜测个人信息。
- 用户只要求排盘时只整理真实结构化结果，不额外编造解读；进一步要求解读时也只依据 `ziwei_calculate` 的真实结果。
- MCP 步骤没有模型调用时成本为 0；结果注入 DeepSeek 后的 Token 和人民币成本只计入对应模型步骤。
- 解绑命语并发布后，平台通用页面和新运行不再带命理专用能力；命理内容只存在于配置、绑定、Trace 和演示对话。
- YIAI Center 不负责从 Git 下载、安装或升级 MCP Server。
- 非白名单 Tool、写 Tool 和未验证 Tool 不得参与运行。
- MCP 业务名称不得进入平台导航、通用标题、Router 固定规则或聊天主流程分支。
